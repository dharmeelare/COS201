How It Works

Bubble Sort repeatedly compares adjacent elements and swaps if they're in the wrong order. Each pass "bubbles" the largest unsorted element to the end. Time complexity: O(n²).

🔵 Low bound🩷 High bound🩵 Mid point🟣 Found!

How It Works

Binary Search compares the target with the middle element. If smaller → search left half; if larger → search right half. Halves the search space each step. Time complexity: O(log n). Data MUST be sorted.

🟡 Checking🟣 Found!
Linear search checks each element one by one. Works on unsorted data.
How It Works

Linear Search checks each element sequentially from the start until it finds the target or exhausts the array. Works on sorted or unsorted data. Time complexity: O(n).

Singly Linked List — each node points to the next, last points to null

Singly Linked List

Each node holds a value and a pointer to the next node. addFirst/addLast are O(1); search is O(n). Dynamic size — no fixed capacity like arrays.

How Recursion Works

Each recursive call creates a stack frame. The call stack grows downward (base case at bottom) then unwinds upward as values are returned.
